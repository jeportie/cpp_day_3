C++ Inheritance Quiz

Test your understanding of inheritance concepts in C++.

Question 1: What is inheritance in C++?

a) A mechanism to create multiple instances of a class
b) A mechanism where a class inherits attributes and methods from another class
c) A way to hide implementation details of a class
d) A technique to overload functions in a class
Correct Answer: b) A mechanism where a class inherits attributes and methods from another class
Inheritance is a fundamental concept in OOP that allows a class to inherit properties and behaviors from another class, promoting code reuse and establishing relationships between classes.

Question 2: In the following code, what is the relationship between ScavTrap and ClapTrap?

class ClapTrap {
    // ...
};

class ScavTrap : public ClapTrap {
    // ...
};
a) ScavTrap is a friend of ClapTrap
b) ScavTrap has a ClapTrap (composition)
c) ScavTrap is a ClapTrap (inheritance)
d) ClapTrap is a ScavTrap (inheritance)
Correct Answer: c) ScavTrap is a ClapTrap (inheritance)
The code shows that ScavTrap inherits publicly from ClapTrap, establishing an "is-a" relationship. This means a ScavTrap is a specialized type of ClapTrap.

Question 3: When a derived class object is created and destroyed, what is the order of constructor and destructor calls?

a) Base constructor, derived constructor, base destructor, derived destructor
b) Derived constructor, base constructor, derived destructor, base destructor
c) Base constructor, derived constructor, derived destructor, base destructor
d) Derived constructor, base constructor, base destructor, derived destructor
Correct Answer: c) Base constructor, derived constructor, derived destructor, base destructor
Construction happens from base to derived (building the foundation first), while destruction happens from derived to base (dismantling from the top down). This ensures that the derived part of an object never accesses a destroyed base part.

Question 4: What does the following constructor do in the ScavTrap class?

ScavTrap::ScavTrap(std::string const name) : ClapTrap(name)
{
    setHp(100);
    setMp(50);
    setAtk(20);
}
a) It creates a ScavTrap with default values
b) It calls the ClapTrap constructor with the name parameter, then sets ScavTrap-specific values
c) It overrides the ClapTrap constructor
d) It creates a ClapTrap object inside the ScavTrap
Correct Answer: b) It calls the ClapTrap constructor with the name parameter, then sets ScavTrap-specific values
The constructor uses an initialization list to call the base class (ClapTrap) constructor with the name parameter. Then, in the body, it sets ScavTrap-specific attribute values that override the ones inherited from ClapTrap.

Question 5: What is method overriding in the context of inheritance?

a) Creating multiple methods with the same name but different parameters
b) Hiding a base class method by declaring a method with the same name in the derived class
c) Making a method inaccessible in the derived class
d) Creating a new method in the derived class that doesn't exist in the base class
Correct Answer: b) Hiding a base class method by declaring a method with the same name in the derived class
Method overriding occurs when a derived class provides its own implementation of a method that is already defined in the base class. This allows the derived class to customize or extend the behavior inherited from the base class.

Question 6: Complete the code to make ScavTrap inherit from ClapTrap and override the attack method:

class ClapTrap {
public:
    void attack(const std::string& target);
};

class ScavTrap : ________ ClapTrap {
public:
    ________ attack(const std::string& target);
};
a) private, virtual void
b) public, void
c) protected, virtual void
d) friend, void
Correct Answer: b) public, void
To make ScavTrap inherit publicly from ClapTrap, we use "public" as the access specifier. To override the attack method, we simply declare a method with the same signature in the derived class. In C++98, the "override" keyword wasn't available, so we just use "void attack(const std::string& target)".

Question 7: With public inheritance, what happens to the access levels of base class members in the derived class?

a) Public members become private, protected members become private, private members become inaccessible
b) Public members become public, protected members become protected, private members remain inaccessible
c) All members become public in the derived class
d) All members maintain their original access levels but can be accessed directly in the derived class
Correct Answer: b) Public members become public, protected members become protected, private members remain inaccessible
With public inheritance, the access levels of base class members are preserved in the derived class, except for private members which are always inaccessible directly from the derived class.

Question 8: Why is the destruction order (derived destructor first, then base destructor) important?

a) It's not important; the order could be reversed without issues
b) To ensure that the derived class can access base class members during destruction
c) To ensure that the derived part of an object never accesses a destroyed base part
d) It's a C++ rule with no practical significance
Correct Answer: c) To ensure that the derived part of an object never accesses a destroyed base part
The destruction order (derived first, then base) ensures that the derived class destructor can still access base class members if needed during its execution. If the base class was destroyed first, the derived destructor might try to access already destroyed base class members, leading to undefined behavior.

Question 9: In our exercise, what is the purpose of the guardGate() method in ScavTrap?

a) To override a method from ClapTrap
b) To provide a unique functionality specific to ScavTrap
c) To access private members of ClapTrap
d) To initialize ScavTrap's attributes
Correct Answer: b) To provide a unique functionality specific to ScavTrap
The guardGate() method is a special ability unique to ScavTrap that doesn't exist in ClapTrap. It demonstrates how a derived class can extend the functionality of its base class by adding new methods.

Question 10: What would happen if we tried to access a private member of ClapTrap directly in ScavTrap?

a) It would work fine because ScavTrap inherits from ClapTrap
b) It would cause a compilation error because private members are not accessible in derived classes
c) It would work but generate a warning
d) It would cause a runtime error
Correct Answer: b) It would cause a compilation error because private members are not accessible in derived classes
Private members of a base class are not accessible directly in derived classes. This is why we use getter and setter methods (like getHp(), setHp()) to access and modify these attributes in the derived class.
